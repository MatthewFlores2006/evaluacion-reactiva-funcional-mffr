



&nbsp;   Comparación de Paradigmas (0.25 puntos): ¿Qué ventajas encontraste al usar programación funcional en este caso de uso? ¿Qué desafíos enfrentaste al evitar la mutación?



&nbsp;	En casos como este la programación funcional por lo general resulta en funciones que son tanto mucho más cortas y que requieren una cantidad mucho menor de trabajo

&nbsp;	manual, por lo tanto en teoría son más fáciles de implementar en varios programas, y aparte de ello que son más legibles para el mantenedor.

&nbsp;	Con respecto a la mutación, en mi caso fue el hecho de tener que evitar muchas prácticas y costumbres que a lo largo del tiempo se han hecho memoria para mí, y esto

&nbsp;	ha resultado en más complicación para poder escribir una solución en un paradigma totalmente distinto y de una forma completamente diferente a la que he estado

&nbsp;	acostumbrado a usar hasta ahora.



&nbsp;   Análisis Crítico (0.25 puntos): ¿En qué situaciones preferirías programación funcional sobre imperativa y viceversa?



&nbsp;	Prefiriese la programación funcional al manipular grandes tablas de datos; Pues la programación funcional provee muchas más

&nbsp;	herramientas incluidas para lidiar con situaciones así, y el paradigma en sí está más concentrado en formas de solucionar problemas

&nbsp;	que están adaptados a la manipulación pero no modificación de datos ya existentes.

&nbsp;	También lo prefiriese para problemas con mayor énfasis matemático, pues nuevamente las funciones usualmente incluidas proveen

&nbsp;	implementaciones nativas para las secuencias de pasos que se necesitan para resolver problemas así; cosas como iteración, secuencias

&nbsp;	matemáticas o patrones son potenciales problemas que soluciona de forma más rápida y eficiente.

&nbsp;	

&nbsp;	Sin embargo a decir verdad hay muchas más ocasiones en las que prefiriese programación imperativa.

&nbsp;	Un ejemplo más obvio fueran los sistemas de un videojuego: Hay muchas cosas de las cuales se deben realizar cambios, 

&nbsp;	más crucialmente los sistemas de este dependen de cambios en variables y atributos, ósea que fundamentalmente dependen de la mutación.

&nbsp;	Sin esta un sistema dónde una interfaz gráfica está en constante cambio, un sistema de física requiere realizar simulaciones continuas o simplemente

&nbsp;	se mantiene una cuenta de varios valores cruciales para la experiencia no funcionaran de la misma forma para nada, y cosas como manipulación de

&nbsp;	datos a base de archivos se complicaría en gran parte. (Al menos, según tengo entendido)

&nbsp;	hace que para mí resulte mucho más fácil de manejar y entender a diferencia de JavaScript, ya que como Java ciertas cosas funcionan
